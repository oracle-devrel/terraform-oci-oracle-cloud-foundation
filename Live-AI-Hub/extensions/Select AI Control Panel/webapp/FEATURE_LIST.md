# ACME AI Agent Web App Feature List

## Overview

ACME is a fictional company used for demonstration purposes. The ACME AI Agent Web App is a Node.js browser application for querying Oracle Database AI features from a controlled web interface. It supports Oracle AI Agent teams, Select AI profiles, RAG knowledge-base workflows, OCI Generative AI post-processing, report management, and admin operations for AI objects.

The ACME AI Agent Web App is a Node.js browser application for querying Oracle Database AI features from a controlled web interface. It supports Oracle AI Agent teams, Select AI profiles, RAG knowledge-base workflows, OCI Generative AI post-processing, report management, and admin operations for AI objects.

## Core User Features

### Prompt Console

- Run natural-language prompts against an Oracle AI Agent team.
- Run direct Select AI prompts against a configured Select AI profile.
- Switch runtime mode between `AI Agent Team` and `Select AI Profile`.
- Configure the active agent team and Select AI profile from the UI.
- Show raw database output for traceability.
- Render detected or generated HTML output inside a sandboxed iframe.
- Save rendered HTML as a report under the app's `public/Reports` folder.
- Upload generated response markdown to OCI Object Storage and rebuild the RAG vector index.

### LLM Rendering

- Send database responses to OCI Generative AI for HTML post-processing.
- Configure default LLM endpoint, model ID, compartment, auth profile, and OCI config file.
- Store and reuse custom LLM rendering instructions.
- Select rendering instructions from the console.
- Update a Select AI profile to use a chosen LLM configuration.

### Authentication

- Login-protected UI and API.
- Session-based authentication.
- Admin and standard user roles.
- Password change flow for signed-in users.
- Admin-only user management for creating users and assigning roles.

## Administration Features

### AI Agents

- View AI agent teams and agents from the configured database.
- Select the default AI agent team.
- Run the latest team task execution workflow.
- Inspect agent metadata returned by the database.

### AI Manager

- Review AI tools, tasks, agents, teams, Select AI profiles, and RAG profiles.
- Delete supported AI objects and profiles from the UI.
- Display metadata tables for AI object inventory.

### AI Builder

- Create DBMS Cloud AI Agent tools.
- Create AI tasks.
- Create AI agents.
- Create AI teams.
- Create Select AI NL2SQL profiles.
- Create RAG profiles backed by configured vector indexes.
- Pre-filled templates help speed up object creation.

### AI Topology

- Load AI topology metadata from the database.
- Generate a visual topology for teams, agents, tasks, tools, functions, and profiles.
- Download the generated topology as a PNG image.

### Select AI Profiles

- List Select AI profiles available to the active database connection.
- Inspect profile metadata from the UI.

### RAG Management

- View RAG pipelines.
- View pipeline attributes.
- View cloud storage object metadata.
- View pipeline history.
- Display configured OCI Object Storage location.
- Rebuild vector indexes after markdown upload.

### Markdown Upload

- Upload markdown FAQ or knowledge files to OCI Object Storage.
- Validate uploaded filenames against the configured RAG object pattern.
- Trigger vectorization after upload.
- Use configured OCI storage location, database credential, RAG profile, and vector index settings.

### Reports Manager

- Browse files and folders under `public/Reports`.
- Preview PDF and HTML reports.
- Create folders.
- Rename folders.
- Delete folders.
- Rename report files.
- Delete report files.
- Move report files between folders with drag and drop.
- Save HTML generated by the prompt console into the selected Reports folder.

### MCP Tester

- Test the configured Oracle Database MCP endpoint from the server.
- Use either `MCP_AUTH_TOKEN` or database username/password authentication.
- Run MCP actions against a selected profile and prompt.
- Display elapsed time and response details.

### User Prompts

- Browse recent user prompts from the configured prompt source.
- Paginate prompt history.
- Display newest prompts first.

### Memory

- Browse saved agent memory records.
- Use the configured `MEMORY_TABLE`.
- Paginate memory records.

### Links

- Manage useful external links.
- Add links from the admin UI.
- Delete links.
- Open saved links in a browser window.

### Runtime Configuration

- Manage active database configuration from the UI.
- Manage active LLM configuration from the UI.
- Support multiple named database configurations in `config/runtime-config.json`.
- Support multiple named LLM configurations in `config/runtime-config.json`.
- Choose default database and LLM configurations.
- Preserve configured secret values when forms are saved with blank secret fields.
- Restart the app from the admin UI after configuration changes.

## Server And API Features

- Node.js HTTP server with no frontend build step.
- Static UI served from `public`.
- REST-style JSON endpoints under `/api`.
- Oracle Database connection pooling through `oracledb`.
- Optional wallet support through `TNS_ADMIN` and `DB_WALLET_PASSWORD`.
- Configurable request timeout and JSON body size.
- Health endpoint at `/api/health`.
- Runtime defaults for `SELECT_AI_PROFILE`, `AI_AGENT_TEAM`, LLM settings, RAG settings, and OCI settings.

## Key Configuration Files

- `.env`: server environment variables and secrets.
- `.auth`: local user store created by the server when missing.
- `config/runtime-config.json`: named database and LLM runtime configurations.
- `config/llm-instructions.json`: reusable LLM rendering instructions.
- `config/links.json`: admin-managed external links.
- `public/Reports`: generated and managed reports.

## Required Runtime Dependencies

- Node.js 20 or newer.
- Oracle Database access with `DB_USER`, `DB_PASSWORD`, and `DB_CONNECT_STRING`.
- Oracle Instant Client or wallet configuration when required by the database.
- OCI configuration for Generative AI, Object Storage, and related workflows.
- Network access from the server to Oracle Database, OCI endpoints, and any configured MCP endpoint.
